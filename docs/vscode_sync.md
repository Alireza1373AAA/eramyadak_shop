# همگام‌سازی تغییرات پروژه با Visual Studio Code

برای اینکه تغییرات جدیدی که روی پروژه اعمال شده‌اند را بدون کپی دستی کدها داخل Visual Studio Code مشاهده کنید، کافی است مخزن را از طریق Git به‌روز کنید. مراحل زیر را دنبال کنید:

## ۰. در صورت نیاز، مخزن را کلون کنید
- اگر پروژه را هنوز روی سیستم خود ندارید، در ترمینال دستور زیر را اجرا کنید و به جای `Alireza1373AAA` و `eramyadak_shop` اطلاعات مخزن مدنظر را قرار دهید:
  ```bash
  git clone https://github.com/Alireza1373AAA/eramyadak_shop
  ```
- سپس وارد پوشهٔ پروژه شوید و آن را در VS Code باز کنید:
  ```bash
  cd eramyadak_shop
  code eramyadak_shop
  ```
- اجرای دستور `code` همان پوشه را در VS Code باز می‌کند؛ اگر کلاینت VS Code در PATH سیستم ثبت نشده است، از منوی Start یا آیکون برنامه برای باز کردن آن استفاده کنید و پوشه را دستی انتخاب نمایید.

1. **مخزن را در VS Code باز کنید.**
   - از منوی «File» گزینهٔ «Open Folder…» را انتخاب کنید و پوشهٔ پروژهٔ `eramyadak_shop` را باز کنید.

2. **به بخش Source Control بروید.**
   - در نوار کناری آیکون Source Control (یا کلید میانبر `Ctrl+Shift+G`) را بزنید.

3. **دریافت آخرین تغییرات از مخزن.**
   - اگر پروژه به یک مخزن Git متصل است، روی دکمهٔ `...` در بالای پنل Source Control کلیک کنید و گزینهٔ `Pull` یا `Pull from...` را انتخاب کنید تا آخرین تغییرات دریافت و روی فایل‌ها اعمال شود.
   - در صورت نیاز می‌توانید از ترمینال داخلی VS Code نیز دستور زیر را اجرا کنید:
     ```bash
     git pull
     ```

4. **بررسی فایل‌های به‌روز شده.**
   - پس از دریافت تغییرات، فایل‌های جدید یا به‌روز شده در Explorer سمت چپ قابل مشاهده هستند و نیازی به وارد کردن دستی کدها نیست.

5. **فعال‌سازی ذخیره و قالب‌بندی خودکار (اختیاری).**
   - برای اطمینان از اینکه قالب‌بندی کدها هنگام ذخیره به‌طور خودکار انجام می‌شود، از منوی `File > Preferences > Settings` گزینهٔ `Format On Save` را فعال کنید.

با طی این مراحل، تمام تغییرات ثبت‌شده در مخزن (شامل پیاده‌سازی سرویس پیامک فراز، مدیریت OTP و فرم ثبت‌نام) مستقیماً در محیط VS Code در دسترس شما خواهند بود بدون اینکه لازم باشد کدها را تک‌به‌تک کپی کنید.
